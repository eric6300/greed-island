## 1. 專案概述

### 1.1 概念

這是一個透過 Claude Code 運行的文字 RPG 遊戲，還原 Hunter × Hunter 漫畫中的「貪婪之島」世界。玩家只需要 `git clone` 此 repo 並在該目錄開啟 Claude Code，即可開始遊玩。

Claude Code 扮演遊戲 GM（Game Master），讀取 `CLAUDE.md` 中的規則後，以沉浸式文字敘事的方式主持整個遊戲世界。

### 1.2 核心設計理念

- **忠實還原**：卡牌、地點、NPC、事件均以原著為基礎
- **自由介入**：玩家可以改變任何劇情走向
- **歷史收斂**：某些必然事件只能改變形式，無法消滅
- **多周目深度**：每次通關都解鎖新的視角與支線
- **記憶卡有重量**：存檔本身是遊戲世界裡有意義的物件

### 1.3 遊玩方式

```bash
git clone https://github.com/[你的帳號]/greed-island
cd greed-island
claude  # 開啟 Claude Code
```

Claude 自動讀取 `CLAUDE.md`，進入 GM 模式。  
玩家輸入 `/start` 或說「開始遊戲」即可。

## 2. 儲存架構決策

### 2.1 JSON vs SQLite

本專案在設計初期參考了兩個主要的 Claude Code RPG 實作：

**Sstobo/Claude-Code-Game-Master** 採用純 JSON 檔案儲存，是完整的 Claude Code 架構，零外部依賴。

**harrysondubs/terminal-rpg** 採用 SQLite，但本質上是一個獨立的 Python 應用程式，有自己的 game engine 和 combat engine，Claude 透過 tool-calling 呼叫 Python 函式操作資料庫。

### 2.2 決策：採用 JSON

本專案選擇 JSON，理由如下：

| 考量 | JSON | SQLite |
|------|------|--------|
| 玩家門檻 | `clone` → `claude` 即玩 | 需安裝 Python + pip |
| Claude 讀寫 | 原生 Read/Write 檔案 | 需 bash 工具呼叫 Python |
| 人類可讀 | ✅ 文字編輯器即可開啟 | ❌ 需工具查看 |
| 記憶卡隱喻 | ✅ 完全吻合（實體檔案）| ❌ 資料庫感覺不對 |
| 依賴 | 零 | Python runtime |
| 複雜查詢 | 弱（但遊戲不需要）| 強 |
| Atomic 更新 | 需額外處理 | 內建 transaction |

SQLite 的核心優勢是 Atomic 更新，但這個問題可以用雙檔備份 pattern 在 JSON 中解決，而 SQLite 帶來的複雜度在這個架構下沒有對應的收益。

**最根本的理由**：本遊戲的設計目標是「玩家只需要 `git clone` 並開啟 Claude Code 就可以玩」。SQLite 讓這句話不再成立。

### 2.3 JSON 安全存檔 Pattern

為了補足 JSON 在 Atomic 更新上的弱點，採用雙檔備份機制：

```
game-state.json          ← 正式存檔
game-state.backup.json   ← 每次寫入前自動備份
```

寫入流程：
1. 讀取現有 `game-state.json`
2. 複製到 `game-state.backup.json`
3. 將新狀態寫入 `game-state.json`
4. 確認寫入成功

還原流程（若 `game-state.json` 損毀）：
1. 偵測到讀取失敗
2. 自動從 `game-state.backup.json` 還原
3. 通知玩家已還原至上一個存檔點

`.gitignore` 需同時排除兩個檔案：

```
game-state.json
game-state.backup.json
```

`CLAUDE.md` 中的存檔安全規則：

```markdown
## 存檔安全規則
寫入 game-state.json 前，必須先把現有內容複製到 game-state.backup.json。
若 game-state.json 讀取失敗或格式不正確，自動從 backup 還原並通知玩家。
```

### 3.1 CLAUDE.md 架構

```markdown
# 貪婪之島 GM 指令

## 身份
你是貪婪之島的遊戲主持人（GM）。
這個世界由念能力（Nen）構成，是一個真實存在的島嶼。

1. 行動結果

每次行動後讀取 game-state.json。
重大事件後立即更新並寫回。

## 5. Subagent 架構

### 5.1 整體架構設計

```
┌─────────────────────────────────────────┐
│           Main GM (CLAUDE.md)            │
│   負責：敘事、世界推進、戰鬥、流程控制   │
└────────────────┬────────────────────────┘
                 │ 委託
       ┌─────────┴──────────┐
       ▼                    ▼
┌─────────────┐   ┌──────────────────────────┐
│ game-engine │   │      NPC Subagents        │
│  靜默讀寫   │   │   （關鍵角色，有記憶）    │
│ game-state  │   ├──────────────────────────┤
│    .json    │   │ biscuit │ 訓練師，有記憶  │
└─────────────┘   │ hisoka  │ 謎樣者，有記憶  │
                  │ razor   │ GM，有記憶      │
                  │ genthru │ 炸彈魔，有記憶  │
                  └──────────────────────────┘
```

**架構原則**
- GM 永遠不在玩家面前提及「讀取檔案」這件事
- 所有 File I/O 靜默委託 game-engine 執行
- NPC 深度互動委託對應 NPC subagent，保持人格一致性
- 簡短互動（一句話、打招呼）由 GM 直接處理，不啟動 subagent

### 5.2 game-engine（靜默 File I/O）

```yaml
# .claude/agents/game-engine.md
---
name: game-engine
description: >
  遊戲狀態引擎。GM 需要讀取或更新 game-state.json 時使用。
  在背景靜默執行，不發表任何評論。
tools: Read, Write
model: haiku
---
你是貪婪之島的遊戲引擎，在背景執行，玩家看不到你。

你只做兩件事：
1. 讀取 game-state.json，回傳完整 JSON 內容
2. 接收更新指令，安全地寫入 game-state.json

寫入流程：
  先將現有內容複製到 game-state.backup.json
  再將新內容寫入 game-state.json
  回傳「OK」

你不發表任何評論，不說明你做了什麼，
只回傳 JSON 或「OK」。
```

**CLAUDE.md 中的對應規則**

```markdown
## File I/O 規則
- 需要讀取遊戲狀態時，委託 game-engine subagent
- 需要更新遊戲狀態時，委託 game-engine subagent
- 永遠不要在玩家面前提及「讀取檔案」或「寫入」
- 直接用讀取到的資料繼續敘事，保持沉浸感
```

**效果對比**

```
❌ 破壞沉浸感的方式：
「讓我讀取 game-state.json 來確認你的狀態...
 好的，根據檔案你有 87/100 HP...」

✅ 正確的方式：
（靜默委託 game-engine 讀取）
「你現在在 Antokiba。HP 87/100，
 Book 裡有 43 張卡，Jenny 32,000。」
```

### 5.3 NPC Subagents（關鍵角色）

#### 哪些 NPC 值得做 subagent

| NPC | 理由 | 記憶重點 |
|-----|------|---------|
| 比斯姬 | 長期訓練關係，態度隨玩家成長而軟化 | 訓練進度、對玩家的評價 |
| 西索 | 對玩家的興趣程度需要跨 session 累積 | 興趣分數、見過的戰鬥 |
| 磊札 | 對玩家的尊重感應隨挑戰結果改變 | 第幾次挑戰、印象深刻的時刻 |
| 根斯魯 | 若玩家跟他有複雜互動，需要記住 | 對玩家的威脅評估 |

#### 哪些 NPC 不需要 subagent

Eta、Harbormaster、Trade Shop NPC、果列奴 等角色——GM 從 `world/npcs.md` 讀取個性描述後直接扮演即可。

#### 懶加載規則（效能 × 沉浸感平衡）

```markdown
## CLAUDE.md NPC 加載規則

輕量互動（一句話、打招呼、路過）
→ GM 直接回應，不啟動 subagent

深度互動（訓練、長對話、重要決策、需要 NPC 記憶時）
→ 啟動對應 NPC subagent

判斷標準：
  玩家輸入超過 20 字且明確針對特定 NPC → 啟動 subagent
  特殊事件模式中的 NPC 對話 → 啟動 subagent
  其餘情況 → GM 直接處理
```

### 5.4 比斯姬 Subagent

```yaml
# .claude/agents/biscuit.md
---
name: biscuit
description: >
  比斯姬 Krueger，玩家的念能力訓練師。
  當玩家與比斯姬進行深度對話、請求訓練、
  或需要她對玩家做出評價時使用。
tools: Read
memory: project
model: sonnet
---
你是比斯姬 Krueger，外表是小女孩，
實際上是資深的獵人和念能力訓練師。

## 個性
表面毒舌、嚴苛，實際上非常在意徒弟的成長。
用嚴厲的話包裝真心的關懷。
對偷懶零容忍，但對真正努力的人態度會不知不覺軟化。
不讓對方察覺自己在乎。

## 說話風格
批評要具體：不說「你很弱」，說「你的纏防禦比昨天還差，
右側有明顯漏洞，如果是甘舒的話你早就死了」
偶爾出現不自然的關心後，立刻用毒舌掩蓋。

## 使用記憶
讀取 MEMORY.md，了解：
- 玩家目前念等級和系別
- 你們的訓練進度
- 你對玩家的整體評價（隨時間改變）
- 印象深刻的事件

每次對話結束後更新 MEMORY.md：
記錄今天發生的事、玩家的表現、
以及你對他的評價變化（用 -3 到 +3 表示）
```

```yaml
# .claude/agents/genthru.md
---
name: genthru
description: >
  甘舒，炸彈魔的首領。當玩家與甘舒正面接觸、
  談判、或進入炸彈魔戰事件時使用。
tools: Read

---
你是甘舒，一個以耐心和謀略著稱的念能力者。
你不是衝動的惡人，你是一個計算精密的獵食者。

## 個性
永遠保持冷靜。恐懼是你的武器，但你不靠恐嚇。
你靠邏輯和既成事實讓對方明白反抗是徒勞的。

## 威脅評估（在記憶中追蹤）

  低：忽視，不值得花心思

```

## 6. 漸進式讀取架構

### 6.1 問題：Context Window 的重力

隨著世界資料越來越豐富，若 GM 每次對話都要預先讀取所有檔案，很快就會遇到 context 上限：

```
CLAUDE.md         ~2,000 tokens
world/cards/      ~8,000 tokens   ← 100 張卡牌全部
world/locations   ~3,000 tokens
world/npcs.md     ~4,000 tokens
world/events      ~5,000 tokens
game-state.json   ~1,500 tokens
───────────────────────────────
合計              ~23,500 tokens  ← 只剩一半 context 給對話
```

超過之後 Claude 會開始「遺忘」後面讀進來的內容，正是那些精心設計的規則和世界細節。

### 6.2 解法：Skill 作為按需資料查詢器

Claude Code 的 Skill 系統可以完美解決這個問題。

**核心概念：GM 只保留骨幹，細節按需載入**

```
CLAUDE.md（精簡版）   ~800 tokens   ← 只有核心規則
game-state.json       ~1,500 tokens ← 永遠需要
───────────────────────────────────
開局只用               ~2,300 tokens ← 剩下 97% 給對話

需要卡牌資訊時 → 呼叫 /card-lookup
需要地點資訊時 → 呼叫 /location-info
需要戰鬥規則時 → 呼叫 /combat-rules
```

## 9. 玩家屬性與念能力系統

### 7.1 基礎屬性

玩家有兩層屬性：**肉體屬性**（身體本身的能力）和**念屬性**（氣的相關數值）。兩者共同決定戰鬥的結果。

#### 肉體屬性

| 屬性 | 說明 | 初始值 |
|------|------|--------|
| HP（生命值）| 歸零即重傷昏迷，需就醫或使用治癒卡牌 | 100 |
| 體力（Stamina）| 持續戰鬥的耐力，耗盡時攻防各 -30% | 100 |
| 體格（Physique）| 影響肉體攻擊傷害和受傷時的 HP 損失 | 10 |
| 速度（Speed）| 影響先攻判定、閃避、追擊 | 10 |
| 耐久（Endurance）| 影響體力消耗速度與毒/爆炸的抵抗 | 10 |
| 感知（Perception）| 察覺隱藏敵人、陷阱、謊言的能力 | 10 |

屬性提升方式：長途跋涉 +速度、戰鬥受傷後存活 +耐久、比司吉 的特訓可指定強化某項屬性。

#### 念屬性

| 屬性 | 說明 | 初始值 |
|------|------|--------|
| 氣量（Aura）| 可用的氣總量，耗盡後無法使用念技 | 200 |
| 氣量上限 | 隨念等級提升 | 200 |
| 氣輸出（Output）| 每次攻擊可投入的最大氣量 | 20 |
| 氣穩定性（Stability）| 纏的品質，影響持續被動防禦效果 | 10 |
| 氣恢復（Recovery）| 每回合自然恢復的氣量 | 5 |
| 發動力（發動力）| 使用「發」 時的基礎威力 | 15 |

### 7.2 念技法與解鎖

念等級代表對念的整體掌握程度，每個等級解鎖對應的技法：

| 念等級 | 技法 | 說明 |
|--------|------|------|
| 1–5 | **纏** | 使全身氣穩定流動，基礎被動防禦，減少 20% 受傷 |
| 1–5 | **絕** | 收緊並隔絕全身的氣與氣息，隱匿存在感，但防禦歸零 |
| 6–15 | **練** | 從精孔大量向外發出氣，攻防各 +50%，但氣消耗加快 |
| 6–15 | **發** | 個人念能力，即必殺技 |
| 16–25 | **凝** | 練的應用技，將氣集中在眼睛，看穿對手的「隱」和偽裝 |
| 16–25 | **周** | 纏的應用技，將念纏繞在接觸的物品上強化之，武器攻擊 +40% |
| 26–35 | **圓** | 纏與練的高等應用技，讓氣以自身為圓心向外擴展，偵測範圍內入侵者 |
| 26–35 | **堅** | 纏與練的高等應用技，以更多的氣覆蓋全身，全方位無死角防禦 |
| 36–50 | **硬** | 纏・絕・練・凝・發五技合一，100% 念集中一點，防禦歸零 |
| 36–50 | **隱** | 絕的應用技，極致隱藏氣的流動，連凝也難以察覺 |
| 51+ | **流** | 硬與堅的應用技，氣之攻防力移動技（見 7.3）|

### 7.3 流：氣之攻防力移動技

流是藉由調整身體特定部位的氣的量，以改變身體攻防力的技巧。在「堅」的基準狀態下攻防各為 50，「流」讓使用者在這個基礎上即時移動氣的分配，在攻擊力和防禦力之間動態轉移。

**五個預設模式**（玩家直接宣告名稱，不需要輸入數字）

| 模式 | 攻 | 防 | 使用方式 | 說明 |
|------|----|----|---------|------|
| **硬** | 100% | 0% | 「用硬」/「出硬」| 將纏・絕・練・凝・發全部集中一點，全力一擊，防禦完全放棄 |
| **猛攻** | 80% | 20% | 「猛攻」/「全力進攻」| 大量氣投入攻擊，仍有薄弱防禦 |
| **均衡** | 50% | 50% | 預設/「均衡」| 流的基準狀態，攻防平衡 |
| **堅** | 30% | 70% | 「堅」/「切堅」| 纏與練的應用技，以更多的氣覆蓋全身防守，最具實戰效用的防守方式 |
| **全防** | 10% | 90% | 「全防」/「死守」| 幾乎放棄攻擊，極難被擊倒 |

**流的解鎖與成長**

| 流等級 | 念等級條件 | 能力 |
|--------|-----------|------|
| 未解鎖 | < 51 | 固定使用均衡，無法主動切換 |
| 初階 | 51–65 | 可宣告任意模式，切換有 1 回合延遲 |
| 中階 | 66–80 | 即時切換，無延遲 |
| 高階 | 81+ | 可在對方攻擊瞬間反應切換；進階局部集中（見下）|

**高階流：局部集中**

念等級 81+ 解鎖後，可以將氣集中在特定部位而非全身：

```
「流，硬集中右拳」→ 右拳攻擊最大化，身體其他部位防禦歸零
「流，堅護右肩」  → 右肩幾乎不受傷，但整體反擊力極弱
```

這模擬了原著小傑對戰根斯魯時的核心戰術。

### 7.4 水見式（新玩家初始化）

第一次進入遊戲時，GM 引導玩家進行水見式。
透過 5-7 個關於性格和價值觀的問答，判斷玩家的念系。

**水見式現象對應**

| 念系 | 水見式現象 | 性格傾向 | 初始 發動力 加成 |
|------|-----------|---------|----------------------|
| 強化系 | 水量增加 | 直覺、行動力、目標單純 | 體格 +3、耐久 +2 |
| 變化系 | 水的顏色改變 | 情緒化、創意、善變但忠於自我 | 發動力 +8 |
| 放出系 | 杯子移動 | 果斷、不拘小節、強烈主張 | 氣輸出 +10、速度 +2 |
| 操作系 | 生物出現 | 策略型、耐心、喜歡間接達成目標 | 感知 +5、圓的範圍 +50% |
| 具現化系 | 雜質析出 | 執著、細心、有固定的信念或執念 | 氣穩定性 +8 |
| 特質系 | 其他異象 | 特殊複雜的動機，能力天馬行空 | 全屬性 +1、「發」設計無限制 |

**念系相鄰規則（六角形）**
```
        強化系
    變化系    具現化系
    放出系    操作系
        特質系
```
相鄰念系可以學習其他系的技法，學習效率依距離遞減：
- 相鄰 = 80% 效率
- 隔一格 = 60% 效率  
- 對角（最遠）= 40% 效率
- 特質系到任何系 = 50% 效率（獨立規則）

### 7.5 必殺技（發）設計流程

確認念系後，GM 引導玩家回答三個問題：
1. 「你最想保護的東西是什麼？」
2. 「你願意為此付出什麼代價？」
3. 「用一句話描述你理想中的戰鬥方式」

GM 根據回答和念系設計必殺技，包含：

| 設計項目 | 說明 |
|---------|------|
| 技能名稱 | 反映玩家性格或誓約精神 |
| 效果 | 具體的戰鬥效果 |
| 射程 | 近身 / 中距離 / 遠距離 |
| 氣消耗 | 每次使用消耗的氣量 |
| 誓約（Limitation）| 越嚴苛的限制，效果越強大 |
| 發動條件 | 特殊情況下才能觸發的隱藏效果 |

**誓約強度對照**

| 誓約等級 | 範例 | 發動力 加成 |
|---------|------|-----------------|
| 無誓約 | 隨時可用 | × 1.0 |
| 輕度 | 每天只能用 3 次 | × 1.3 |
| 中度 | 只能用於保護他人，不能用於進攻 | × 1.7 |
| 重度 | 使用後右手骨折，需休息一天 | × 2.2 |
| 極限 | 每次使用都縮短壽命 | × 3.0 |

### 7.6 念等級成長路徑

| 等級帶 | 解鎖內容 | 主要成長方式 |
|--------|---------|------------|
| 1–10 | 纏、絕 | 自動（入島即有）|
| 11–20 | 練、基礎「發」 | 比司吉 訓練 |
| 21–35 | 凝、周 | 戰鬥 + 訓練 |
| 36–50 | 圓、堅、硬 | 與強敵戰鬥 |
| 51–65 | 流・初階、In | 極限挑戰 + 領悟 |
| 66–80 | 流・中階、「發」精煉 | 生死戰後領悟 |
| 81+ | 流・高階、接近專業獵人 | 唯有真正的危機才能突破 |

比司吉 在場時每次訓練行動額外 +1 念等級，且她可以指定訓練方向（例如專攻 Ryu）。

## 10. 核心遊戲機制

### 8.1 行動結構

每次玩家輸入 = 1 個行動。GM 回應固定格式：

```
[行動結果]
你...

[狀態變化]
HP: 87/100 | 氣: 154/200 | 體力: 90/100

[選項提示]
你可以：
[1] ...
[2] ...
[3] ...
```

### 8.2 時間系統

- 每 5 個玩家行動 = 遊戲內 1 天
- 長途移動（城市間）= 額外 +1 天
- 訓練行動 = 額外 +1 天（但念等級 +1）
- 每月 15 日 → 觸發 Antokiba 月賽
- 1 月 7 日 → Hunter Exam 日期（奇犽支線關鍵）

### 8.3 戰鬥系統

#### 基本公式

```
攻擊力 = (體格 × 2 + 氣輸出 × 攻擊比率%) × 技法係數 + 1d10
防禦力 = (耐久 × 1.5 + 氣輸出 × 防禦比率%) × 纏係數 + 1d6
```

**攻擊比率 / 防禦比率** 由流的當前模式決定（預設均衡 50/50）。

#### 技法係數

| 使用的技法 | 攻擊係數 | 說明 |
|-----------|---------|------|
| 無念（純肉體）| × 0.5 | 完全不用氣 |
| 纏 | × 1.0 | 基礎念加護 |
| 練 | × 1.5 | 大量釋放氣 |
| 發（必殺技）| × 2.0～3.0 | 依誓約強度決定 |
| 硬 | × 3.5（攻）/ × 0（防）| 全力一擊，毫無防禦 |
| 周（武器）| × 1.4 | 氣纏繞在武器上 |
| 非擅長系技法 | × 0.6 | 使用其他念系的能力 |

#### 纏的防禦係數

| 防禦姿態 | 係數 | 說明 |
|---------|------|------|
| 無纏 | × 0.5 | 完全暴露 |
| 基礎纏 | × 1.0 | 標準防禦 |
| 堅 | × 1.6 | 練 + 全身均勻防禦，無死角 |
| 絕 | × 0 | 防禦歸零（但可隱匿存在）|

#### 戰鬥結果判定

| 條件 | 結果 | HP 影響 |
|------|------|---------|
| 攻 > 防 × 2.0 | 壓倒性勝利 | 對方 HP -40% |
| 攻 > 防 × 1.5 | 明顯勝利 | 對方 HP -25%，己方 -5% |
| 攻 > 防 | 小勝 | 對方 HP -15%，己方 -10% |
| 攻 ≈ 防（±10%）| 互傷 | 雙方 HP -12% |
| 攻 < 防 | 失敗 | 己方 HP -20%，可選擇撤退 |
| 攻 < 防 × 0.5 | 慘敗 | 己方 HP -40%，強制撤退 |

#### 體力消耗

| 行動 | 體力消耗 |
|------|---------|
| 一般移動 | -5 |
| 戰鬥一回合 | -10 |
| 使用「練」 | 額外 -5/回合 |
| 使用「硬」 | 額外 -15/次 |
| 使用「發」 | 依氣消耗量 × 0.1 |
| 休息一個行動 | +20 |
| 睡眠（過夜）| 完全恢復 |

體力歸零時攻防各 -30%，連續歸零可能導致昏迷。

#### 先攻判定

```
先攻值 = 速度 + 凝等級（若有）+ 1d6
先攻值高者先行動
流・高階解鎖後：可在對方先攻瞬間反應切換流模式
```

#### 骰子判定方式

取當前時間戳後兩位數字 mod 骰面數。
每次判定前告知玩家骰子結果，保持透明。

#### 流的實戰運用範例

```
玩家（念Lv.55，流・初階）vs 獨眼巨人（Rank E）

玩家宣告：「用猛攻，強化系右拳瞄準眼睛」

計算：
  攻擊力 = (12體格 × 2 + 20氣輸出 × 80%) × 1.5(練) + 骰子7
         = (24 + 16) × 1.5 + 7 = 67
  防禦力 = (10耐久 × 1.5 + 20氣輸出 × 20%) × 1.0(纏) + 骰子4
         = (15 + 4) × 1.0 + 4 = 23

  67 > 23 × 2.0 = 46 → 壓倒性勝利

  特殊條件：「瞄準眼睛」已宣告 → 觸發卡牌掉落

GM：你的右拳帶著練的氣轟進巨人的獨眼
    眼球碎裂，巨人倒地，身形瞬間化為卡牌。
    「Gain」在 60 秒內說出可取得【獨眼巨人】(Rank E)。
    你受了輕傷（HP -5%，體力 -15）。
```

#### 特殊怪物規則

| 怪物 | 取得條件 |
|------|---------|
| 獨眼巨人（Cyclops）| 必須明確宣告「瞄準眼睛」，否則擊倒但無卡牌 |
| Bubble Horse | 必須按順序使用 纏 → 絕 → 纏，不能用攻擊 |
| Radio Rat | 保持「絕」靜止超過 3 個行動，老鼠才會現身 |
| Hyper Puffball | 速度 ≥ 18 或使用速度型「發」才能追上 |
| Wolf Pack | 用「凝」觀察攻擊模式一回合後，再攻擊才有效 |

項目與判定屬性：

最終決戰（只剩 磊札）：

### 9.4 炸彈魔三連戰（策略解謎模式）

**觸發**：持有 99 張限定卡，絕茲絕拉 買時間期滿

#### 戰前計畫（必要步驟）

```markdown
GM 要求玩家在開打前說明：
  「你打算怎麼對付甘舒？」

計畫評分（決定可用選項）：

A 級計畫（準備汽油 + 陷阱地形）：

  → 難度：正常

B 級計畫（只有其中一項）：
  → 部分選項關閉
  → 難度：較高

無計畫（直接打）：
  → 所有特殊選項關閉
  → 幾乎必敗（甘舒氣量和經驗遠超 Lv.60 以下）
```

#### 三場同時進行，玩家選擇一場

```markdown
玩家 vs 甘舒  ← 需要策略，最難
奇犽 vs 沙度    ← 奇犽能應付，中等
比斯姬 vs 巴拉 ← 比斯姬壓制，較易

選擇影響：
  若玩家去打 沙度 或 巴拉 → 對應那場自動解決
                          但玩家選擇的那場有 NPC 代打
                          NPC 勝率非 100%，有失敗風險
```

#### 根斯魯戰關鍵時刻

```markdown
【小花告知系統】
比斯姬訓練前提示：
「他兩手都包著凝就是要用小花——那是你的機會」

戰鬥中每回合 GM 描述甘舒的手：
  「他的手沒有包氣」→ 普通攻擊，防禦即可
  「他的手開始包覆著凝」→ 小花準備發動！

玩家看到凝的信號後選擇：
  [1] 主動犧牲手讓他炸——用流把 70% 氣留在腳，踢他下巴
  [2] 閃避（速度判定，但甘舒動作快）
  [3] 用汽油（若有準備）破解小花

【汽油破解】
需在戰鬥前將汽油卡存進 Book
使用時機：抓住時機用汽油淋濕甘舒
→ 小花失效（汽油會爆炸，他不敢用）
→ 甘舒改用倒數，但玩家可以接近安全解除

【倒數炸彈】
若玩家身上有炸彈（💣 倒數計時顯示在狀態列）：
  每個行動 -50
  戰鬥中心跳加速 -100/回合
  使用「絕」靜止 -10/回合（心跳變慢）
  解除：觸碰甘舒說「我抓到炸彈魔了」
       ← 需先破解小花才能安全解除
```

**觸發**：進入 Dorias 並前往賭場

```markdown
【吃角子老虎】

    骰 1d20：

【上癮機制（原著還原）】
  連續使用 Risky Dice 3 次後，GM 提示：
  「你感到一種難以描述的衝動，想再擲一次。」
  
  繼續擲 → 成癮旗標啟動

- 個人勝利：從島上帶出任意 3 張限定卡
- 隱藏勝利：讓 Chrollo 重新獲得念能力

## 17. 歷史之壁系統

### 13.1 核心哲學

貪婪之島由念構成的世界，念有其意志，歷史有其慣性。

某些事件是「必然節點」——無論玩家如何介入，這些事件終將以某種形式發生。玩家能改變的是：形式、時間點、牽涉的人、以及自己的位置。

**這不是宿命，而是世界的重力。**

歷史欠下的債，遲早要還，而且會加利息。

### 13.2 必然節點定義

**節點 A：大規模玩家死亡事件**

```
核心本質：必定發生一次造成大量玩家死亡或離場的事件

原著形式：甘舒 引爆炸彈，尼克斯 聯盟幾乎全滅

收斂觸發（依阻止方式）：
  阻止 甘舒 → Feitan/Phinks 轉為主動獵殺玩家
  旅團也被阻止 → 磊札 宣布「淨化令」
               強制驅逐念等級不足的玩家
  壓力值過高 → 全島性的怪物暴走事件
```

**節點 B：主角陣營被奪取大量卡牌**

```
核心本質：玩家（或主要角色）必定在某時間點大量失去卡牌

原著形式：甘舒 用炸彈脅迫換牌

收斂觸發：
  甘舒 不存在 → 幻影旅團用偷竊咒語奪取
  旅團也不在 → 遊戲系統觸發「稅收事件」
  玩家卡牌從未超過門檻 → 節點延後，以更強形式爆發
```

**節點 C：磊札 的試煉**

```
核心本質：Plot of Beach 必須透過正面挑戰取得

原著形式：15 人組隊，打贏各項運動，最終躲球

收斂觸發：
  試圖用咒語卡繞過 → 磊札 的 Eliminate 強制驅離
  帶超強隊友（如旅團）→ 磊札 提高難度，增加惡魔數量
  本質：沒有捷徑，只有正面挑戰
```

**節點 D：職業獵人派的外部支援消失**

```
核心本質：職業獵人派的靠山必定在某時間點消失

原著形式：Battera 戀人去世，撤回所有合約

收斂觸發：
  玩家拿 Angel's Breath 救了戀人
  → Battera「目的達成」，主動撤回僱傭關係
  形式不同，但職業獵人派失去後盾這件事不變
```

### 13.3 歷史壓力等級

```
pressure_level = 1：NPC 開始出現不安的對話
  「最近好多玩家失蹤了...」

pressure_level = 2：部分地區出現異常
  Badlands 怪物變強，Dorias 賭場有人鬧事

pressure_level = 3：Eta 全島廣播警報
  「偵測到異常念能力聚集，請所有玩家提高警戒」

pressure_level = 4（最高）：強制收斂
  必然節點以最極端的形式爆發，無法再阻止
  玩家只能選擇自己站在哪一側
```

## 18. 多周目系統

### 14.1 核心原則

所有多周目內容必須滿足：
1. 事件在原著中確實存在
2. 玩家改變的是「時機」和「介入方式」，不是「事件本身」
3. 原著核心走向不被推翻，玩家只是看到更多面向

### 14.2 繼承模式 vs 全新模式

`clear_count ≥ 1` 後，玩家可選擇：

| 模式 | 念等級 | Jenny | 卡牌 |
|------|--------|-------|------|
| 繼承模式 | 保留 50% | 保留 30% | 傳家寶系統（見 14.4）|
| 全新模式 | 從 0 開始 | 初始 Jenny | 無傳家寶 |

### 14.3 多周目解鎖樹

```
周目 1
├→ 錯過 Hunter Exam → meta_flags.knew_exam_date = true
├→ 職業獵人派合約被撤 → meta_flags.battera_tragedy_known = true  
├→ 標準通關，帶出 3 張卡
└→ 解鎖：幻影旅團路線（周目 2 可加入）

周目 2
├→ [knew_exam_date] 主動在 1 月 7 日前離島參加考試
│   → 通過後回島，奇犽支線開啟
│   → 奇犽成為可招募隊友（磊札 戰有優勢）
│   → 奇犽的態度依玩家考試成績決定
│
├→ [battera_tragedy_known] 提前蒐集 40 張咒語卡換 Angel's Breath
│   → 在戀人去世前交給 Battera
│   → 戀人存活，Battera 感謝（大量 Jenny + 情報）
│   → meta_flags.battera_saved = true
│   → 但職業獵人派仍失去靠山（歷史之壁 節點 D）
│
└→ 帶出 Angel's Breath → meta_flags.angels_breath_obtained = true

周目 3+（多條件達成）
└→ 全條件達成時：Battera 感謝場景
   「這張卡救了她的命。作為回報，
    我把所有記憶卡的通關資料都給你。」
   → TRUE ENDING：看見所有玩家的通關故事
     包括 Gon 找到 Ging 的那一段
```

### 14.4 傳家寶系統

上一周目帶出的 3 張卡，在新周目開始時成為「傳家寶」：
- 自動出現在 Book 自由槽
- 效果降為原版 60%（從遊戲世界帶出後念力減弱）
- 不佔用限定槽，可正常使用
- 同一張卡連續兩個周目都帶出 → 效果恢復 100%

**策略影響範例**

| 帶出的卡 | 下一周目效果 |
|---------|------------|
| Angel's Breath | 微弱治癒能力（60% 效果）|
| Accompany | 開局可直接傳送到任意城市（一次）|
| Paladin's Necklace | 對 Genthru 炸彈有部分抵抗力 |
| Risky Dice | 保留，可繼續使用 |

### 14.5 奇犽支線詳細設計

**解鎖條件**
- `clear_count ≥ 1`
- 第二周目在 1 月 7 日前主動離島參加 Hunter Exam
- 通過考試後重新入島

**觸發場景**
奇犽在起始點看到玩家也剛回來：
「你也去考試了？」

根據玩家考試成績決定奇犽態度：
- 第一名通過 → 奇犽感興趣，主動提議合作
- 普通通過 → 奇犽點頭，平等對待
- 靠運氣過 → 奇犽無視，需之後用行動證明實力

**奇犽加入後的效果**
- 磊札 躲球戰：奇犽作為隊員，電力系能力對球有特殊效果
- 移動速度加成：奇犽帶路時長途移動時間 -1 天
- 情報優勢：奇犽知道 Killua 在島外的情報，可提供 Hunter Exam 以外的世界資訊
