# 貪婪之島 — 多周目系統

## 周目結束條件

1. **通關**：蒐集 100 張限定卡 + 問答勝出 + 進入利梅羅 + 選擇帶出 3 張卡
2. **死亡**：HP 歸零（Roguelike 永久死亡）
3. **記憶卡賣出**：接受神秘買家的出價（特殊結局）

## 繼承模式（clear_count ≥ 1 時可選）

| 項目 | 繼承 |
|------|------|
| 念等級 | 全部繼承 |
| Jenny | 全部繼承 |
| 肉體屬性 | 全部繼承 |
| 卡牌 | 無（除傳家寶）|
| 傳家寶 | 上周目帶出的 3 張卡（100% 效果）|
| meta_flags | 全部保留 |
| clear_history | 全部保留 |
| NPC 關係 | 重置 |
| 故事旗標 | 重置 |
| 歷史之壁 | 重置 |

## 全新模式

- 一切從零開始
- 唯一保留：memory_card 區段（meta_flags、clear_history）

## Meta Flags 解鎖樹

### 周目 1
- 錯過 Hunter Exam → `knew_exam_date = true`
- 職業獵人派合約被撤 → `battera_tragedy_known = true`
- 標準通關，帶出 3 張卡
- 解鎖：幻影旅團路線（周目 2 可加入）

### 周目 2
- `[knew_exam_date]` → 主動在 1 月 7 日前離島參加考試
  - 通過後回島，奇犽成為可招募隊友
  - 奇犽態度依玩家考試成績決定
- `[battera_tragedy_known]` → 提前集齊 40 張咒語卡換 大天使的氣息
  - 在戀人去世前交給巴特拉 → `battera_saved = true`
  - 但職業獵人派仍失去靠山（歷史之壁 節點 D）
- 帶出 大天使的氣息 → `angels_breath_obtained = true`

### 使用同行指定「金」

#### 條件未達成時（Normal Ending）
使用同行咒語卡指定「金」→ 被傳送到凱特（Kite）的所在地

- 凱特：「你在找金？⋯⋯你找錯人了。不過，如果你是被同行送到我這裡的，代表你走的路還不夠遠。」
- 凱特會透露一些關於金的線索和玩家目前還欠缺什麼的暗示
- `ging_found` 維持 false
- 這個結局可以在任何周目觸發，但不會解鎖任何 meta_flags

#### 條件全部達成時（TRUE ENDING）
**前置條件**：所有 meta_flags 為 true（knew_exam_date、battera_saved、angels_breath_obtained、killua_route_completed）

使用同行咒語卡指定「金」→ 觸發 TRUE ENDING

- 同行卡帶你飛越大海，離開貪婪之島
- 你抵達一個意想不到的地方——金一直在那裡等著
- 金：「你花了很長時間才到這裡。不過，這條路本來就不是用走的，是用活的。」
- 回顧你所有周目的旅程——每一次死亡、每一次選擇、每一個被你改變的人
- `true_ending_unlocked = true`
- `ging_found = true`
- 解鎖 **GM 模式**

## GM 模式（TRUE ENDING 獎勵）

**解鎖條件**：`true_ending_unlocked = true`

金創造了貪婪之島。現在輪到你了。

### 概念

找到金之後，他教你一件事：「這座島不是用來破關的，是用來創造的。」
下一周目開始時，Eta 會多一個選項：「你想以玩家身份進入⋯⋯還是以 **製作者** 的身份？」

### GM 模式下的新能力

選擇 GM 模式後，玩家身份從「冒險者」轉為「島的製作者」。新增 `/gm` 指令群：

| 指令 | 功能 |
|------|------|
| `/gm card [名稱]` | 設計一張全新的限定卡或自由卡（描述效果，系統自動分配編號和等級）|
| `/gm npc [名稱]` | 創造一個新 NPC（設定性格、對話風格、出沒地點）|
| `/gm event [名稱]` | 設計一個新的特殊事件（設定觸發條件、規則、獎勵）|
| `/gm location [名稱]` | 創造一個新地點（設定描述、設施、連接、可取得卡牌）|
| `/gm rule [規則]` | 修改遊戲規則（調整戰鬥公式、時間流速、遭遇率等）|
| `/gm monster [名稱]` | 設計一隻新怪物（設定屬性、取得條件、卡牌效果）|
| `/gm test` | 以玩家視角測試你的修改（暫時切回玩家模式體驗）|
| `/gm publish` | 將你的修改寫入 world/ 檔案，成為永久的遊戲內容 |

### GM 模式的限制

- **不能刪除原著內容** — 只能新增或修改，原著的 100 張限定卡和核心劇情保持不變
- **歷史之壁仍然生效** — 你是製作者，但念的意志不受你控制。必然節點依然存在
- **NPC 有自我意識** — 你可以創造 NPC，但他們被創造後會有自己的意志（subagent 記憶系統），不完全聽從你
- **磊札會審視你** — 作為同為製作者的磊札，會評價你的設計。太離譜的修改會被磊札質疑

### GM 模式的敘事意義

- 原著中，金和他的夥伴花了數年建造這座島
- 玩家經歷多周目後，對島上的每個角落都瞭若指掌
- GM 模式代表：你不只是通關了，你**理解**了這座島
- 金：「享受這個過程。不是只有到達終點才有意義。」

### game-state.json 新增欄位

```json
"gm_mode": {
  "unlocked": true,
  "active": false,
  "created_cards": [],
  "created_npcs": [],
  "created_events": [],
  "created_locations": [],
  "rule_overrides": {},
  "razor_approval_score": 0
}
```

`razor_approval_score`：磊札對你的設計的評價。太低的話，磊札會找你「談談」。
